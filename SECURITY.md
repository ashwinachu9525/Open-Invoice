# Security Policy

## Supported Versions

Currently, only the latest version of Open-Invoice is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

We take the security of Open-Invoice seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to our security team at `security@open-invoice.example.com`.
Please include the following details in your report:
- A description of the vulnerability.
- Steps to reproduce the issue.
- The version of Open-Invoice you are using.
- Any potential impact.

We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress. If you're reporting a critical issue, we will prioritize it accordingly.

## Multi-Factor Authentication (MFA)
Open-Invoice supports Time-Based One-Time Passwords (TOTP) for Multi-Factor Authentication. We strongly encourage all users, especially business owners, to enable MFA in their Account Settings to protect their invoice and customer data.

## Responsible Disclosure
If you find a vulnerability, we ask that you give us a reasonable amount of time to respond to the issue before making it public. We appreciate your effort in making Open-Invoice more secure!
